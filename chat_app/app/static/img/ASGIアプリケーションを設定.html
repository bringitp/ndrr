以下は、ASGI（Asynchronous Server Gateway Interface）を使用してFastAPIアプリケーションを書き換えるためのコード例です。FastAPIはデフォルトでASGIをサポートしているため、特別な変更は必要ありません。ただし、ファイルの構造やインポートなど、いくつかの微調整が必要かもしれません。

  1. まず、FastAPIアプリケーションのコードをapp.pyという名前のファイルに移動します。

  2. main.pyの内容は以下のように書き換えます：

python
import os
from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware
from fastapi.staticfiles import StaticFiles
import uvicorn

# FastAPIアプリケーションをインスタンス化
app = FastAPI()

# CORS設定を行う
app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],  # ここに許可したいオリジンを指定
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)

# main.pyのディレクトリパスを取得
current_dir = os.path.dirname(os.path.abspath(__file__))
# /chat_app/static/ を /main.pyのディレクトリ/static/ にマウント
app.mount("/static", StaticFiles(directory=os.path.join(current_dir, "static")), name="static")

# ルーターをアプリに追加
app.include_router(root_router)
app.include_router(rooms_get_router)
app.include_router(users_endpoints)
# app.include_router(room_condition_get_router)
app.include_router(room_message_get_router)
app.include_router(room_message_post_router)
app.include_router(room_endpoints)

if __name__ == "__main__":
    uvicorn.run(app, host="0.0.0.0", port=7777)

これで、FastAPIアプリケーションはASGIアプリケーションとして実行できるようになりました。ASGIサーバー（例：uvicorn）を使用して実行するために、main.py内のアプリをインポートし、そのファイルを実行します。

ASGIサーバーを起動するには、以下のコマンドを使用できます：

css
uvicorn main:app --host 0.0.0.0 --port 7777

ここでmainはファイル名（main.py）から取得したモジュール名です。上記のコマンドを実行すると、FastAPIアプリケーションがASGIサーバー上で実行され、0.0.0.0:7777でリクエストを受け付けます。

SQLクエリの実行回数を計測するために、通常の方法としてはデータベースのクエリログを活用することが一般的です。ただし、データベースの種類によってはクエリログを有効にする方法が異なります。以下は、一般的なデータベースでのクエリログの有効化方法の例です。

  1. MySQLの場合:

    MySQLでは、クエリログを有効にするには次の手順を実行します。

    a. MySQL設定ファイル（通常はmy.cnfまたはmy.ini）を開きます。

    b. [mysqld] セクションに以下の行を追加します。

    c
    log = /var/log/mysql/mysql.log

    クエリログの出力先ファイルパスを指定しています。適切なディレクトリとファイル名を指定してください。

    c. MySQLサービスを再起動します。

    sudo systemctl restart mysql

  2. PostgreSQLの場合:

    PostgreSQLでは、クエリログを有効にするには次の手順を実行します。

    a. PostgreSQL設定ファイル（通常はpostgresql.conf）を開きます。

    b. logging_collector オプションを on に設定します。

    csharp
    logging_collector = on

    c. データベースを再起動します。

  3. SQLiteの場合:

    SQLiteはファイルベースのデータベースエンジンであり、通常はクエリログを提供しません。代替方法として、アプリケーションのコード内でクエリをカウントすることができます。アプリケーションでクエリを発行するたびに、カウンターを増やすロジックを追加します。

  4. SQL Serverの場合:

    SQL Serverでは、SQL Server Profilerを使用してクエリログをキャプチャできます。SQL Server Management Studio（SSMS）からProfilerを起動し、適切なイベントを設定してクエリログをキャプチャします。

データベースエンジンと設定によっては、クエリログを有効にする方法が異なるため、使用しているデータベースに合わせて適切な方法を選択してください。

クエリログを有効にしたら、実際のベンチマークテストを実行し、クエリログから実行されたSQLクエリの数を抽出できます。ベンチマークツールを使用することで、アプリケーションのパフォーマンスを評価するのに役立ちます。

FastAPIのベンチマークを取得するために、以下のステップに従うことができます。

  1. ベンチマークツールをインストール:

    FastAPIのベンチマークを取得するためには、ベンチマークツールをインストールする必要があります。代表的なベンチマークツールの1つは、Apache Bench（ab）です。以下のコマンドでインストールできます（Linuxの場合）。

    arduino
    sudo apt-get install apache2-utils

    または、Homebrewを使用してMacでインストールすることもできます。

    brew install httpd-tools

  2. FastAPIアプリケーションの起動:

    ベンチマークを実行する前に、FastAPIアプリケーションを実行しておきます。FastAPIアプリケーションのファイル（例：main.py）があるディレクトリに移動し、次のコマンドでアプリケーションを起動します。

    css
    uvicorn main:app --host 0.0.0.0 --port 8000

    このコマンドは、FastAPIアプリケーションを0.0.0.0:8000で実行します。

  3. ベンチマークを実行:

    ベンチマークツール（ab）を使用してFastAPIアプリケーションに対してリクエストを送信し、パフォーマンスを計測します。以下は、abを使用した例です：

    arduino
    ab -n 1000 -c 10 http://localhost:8000/

    このコマンドは、1000回のリクエストを10の同時接続で行います。http://localhost:8000/の部分は、FastAPIアプリケーションのエンドポイントに置き換えてください。

  4. ベンチマークの結果を解析:

    abの実行後、ベンチマークの結果が表示されます。重要なメトリクスとしては、リクエストの完了時間、平均応答時間、リクエストの成功数、エラー数などがあります。これらの結果を解析して、FastAPIアプリケーションのパフォーマンスを評価できます。

FastAPIのベンチマークを取得するには、abを使用する方法が簡単で効果的ですが、他にもさまざまなベンチマークツールが利用可能です。必要に応じて、適切なツールを選択して利用してください。また、リアルな負荷テストを行う際には、アプリケーションの実際の使用ケースに合わせたリクエストを作成することも重要です。

abを使用してFastAPIアプリケーションに対してベンチマークを実行するために、以下のコードをabに対応した形に変更します。

bash
ab -n 1000 -c 10 -H "Authorization: Bearer YOUR_TOKEN" http://localhost:7777/room/1/messages

このコマンドの要点は次のとおりです：

  • -n 1000: 1000回のリクエストを送信します。必要に応じてリクエスト回数を調整できます。
  • -c 10: 同時に10つのリクエストを送信します。同時接続数を調整できます。
  • -H "Authorization: Bearer YOUR_TOKEN": リクエストヘッダーにトークンを含めます。YOUR_TOKENを実際のトークンに置き換えてください。
  • http://localhost:7777/room/1/messages: テスト対象のFastAPIエンドポイントのURLを指定します。必要に応じてURLを変更してください。

このコマンドを実行すると、FastAPIアプリケーションに対して指定されたリクエスト数と同時接続数でベンチマークが実行されます。ベンチマークの結果には、リクエストの完了時間、平均応答時間、リクエストの成功数、エラー数などが含まれます。

コマンドを実行する前に、FastAPIアプリケーションを起動しておくことを忘れないでください。また、リクエストのトークンやURLなどを適切に設定してください。
